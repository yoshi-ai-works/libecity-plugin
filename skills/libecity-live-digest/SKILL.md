# リベシティ 学長ライブまとめ → Googleドライブ保存スキル

## 概要

リベシティの学長ライブまとめページにブラウザでアクセスし、IT・Claude・AI関連トピックをClaudeが**自分の言葉で再構成**してGoogleドライブに保存する。

**コンテンツ取り扱い原則：**
- 原文の全文コピー・長文引用は行わない
- 取得した情報は必ず「要約・分析・言い換え」に変換する
- 直接引用は1箇所100〜200文字以内に限定し、引用符（「」）を付ける
- 個人情報・機密情報（会員の名前・質問内容の固有名詞）は匿名化する
- 出力はリベシティ会員としての個人的な学習メモの位置づけ

---

## トリガーワード例

- 「ライブまとめ送って」
- 「学長ライブのIT・Claude関連まとめて」
- 「リベシティのライブ要約をファイルに保存して」
- 「最新のライブまとめ教えて」

---

## 事前設定（初回のみ）

以下のファイルをユーザー自身で準備する：

| 項目 | パス | 内容 |
|---|---|---|
| `.env` | `~/playwright-mcp/.env` | `LIBECITY_ID=メールアドレス`<br>`LIBECITY_PW=パスワード` |
| `session.json` | `~/playwright-mcp/session.json` | 初回は `{"savedAt": "2000-01-01T00:00:00Z"}` で作成 |
| Google DriveフォルダID | スキル内で設定 | 保存先のGoogleドライブフォルダID |

> **注意：** `.env` と `session.json` は個人情報を含むため、Gitに含めないこと。

---

## 既知の設定値

| 項目 | 値 |
|---|---|
| ライブまとめURL | `https://libecity.com/room_list?room_id=LIVE-Text` |

## 実行フロー

### Step 0: 事前チェック（重複防止）

スキル開始直後に、最新の学長ライブ日付のファイルがGoogle Driveにすでに存在するか確認する。

```
ファイル名: YYYY-MM-DD_学長ライブまとめ.md（ライブ配信日の日付）
```

**⚠️ ファイル名の日付は「今日の日付」ではなく「ライブ配信日」を使う。**
ライブ配信日は投稿タイトル（例: `2026/04/16【学長ライブ】`）から読み取る。
Step 3でライブ日付が確定してから重複チェックを行う。

**重複チェックのクエリ（`parents`フィールドはサポート外のため使わない）：**
```
title = 'YYYY-MM-DD_学長ライブまとめ.md'
```

- **存在する場合：** 「○月○日のライブまとめはすでに保存済みです。上書きしますか？」とユーザーに確認してから続行する
- **存在しない場合：** そのまま次のステップへ

---

### Step 1: リベシティにログイン

ログイン情報はファイルから自動で読み込む。**ユーザーにID/PWを聞かない。**

**ログインの流れ：**

1. `~/playwright-mcp/session.json` を読み込み、`savedAt` から経過時間を確認する
   - 1時間以内 → セッション有効とみなし、`https://libecity.com/sign_in` に遷移後にホームへ到達するか確認
   - 1時間超 or ホームへ到達しない → ID/PWで再ログインへ
2. **ID/PWでのログイン手順：**
   - `~/playwright-mcp/.env` から `LIBECITY_ID` と `LIBECITY_PW` を読み込む
   - `https://libecity.com/sign_in` にアクセス
   - メールアドレス欄・パスワード欄に入力してログインボタンをクリック
   - URLが `https://libecity.com/mypage/home` になったことを確認
3. **ログイン成功後：** `session.json` の `savedAt` を現在時刻に更新する（次回再利用のため）
4. ユーザーに「ログイン完了。○月○日のライブまとめを取得します」と伝える

---

### Step 2: ライブまとめページを開く

1. ログイン後、**3〜8秒のランダム待機**を入れる
2. `https://libecity.com/room_list?room_id=LIVE-Text` に移動する
3. ページタイトルに「学長ライブまとめチャット」が含まれることを確認する
4. さらに **2〜5秒のランダム待機**を入れる

---

### Step 3: 最新の【学長ライブ】投稿を特定する

**ページ構造：**
- チャットメッセージは `overflow:auto` の無名DIVコンテナ内にある
- 各投稿の本文は `.post_text.is_limited.is_close` クラスのDIV
- 「続きを読む」ボタンは `span.post_text_open` クラス

**手順：**

1. JavaScriptでチャットコンテナを最下部までスクロールする
   ```js
   const el = Array.from(document.querySelectorAll('div'))
     .find(e => e.className === '' && 
                window.getComputedStyle(e).overflowY === 'auto' && 
                e.scrollHeight > 10000);
   if (el) el.scrollTop = el.scrollHeight;
   ```

2. 最下部の投稿タイトルを取得し、`YYYY/MM/DD【学長ライブ】` 形式であることを確認する
   - **【限定ライブ】は対象外**。その場合はひとつ上の投稿を確認する

3. 対象投稿の `span.post_text_open`（続きを読む）をクリックして本文を展開する

4. 3秒待機後、`.post_text` の `is_close` クラスが外れたことを確認する

5. ユーザーに「○月○日のライブまとめを取得します」と伝える

---

### Step 4: トピックを読み取る（全文コピー禁止）

**取得の原則：読む・分析する。そのまま書き写さない。**

1. 展開された投稿内の `.post_text` の `innerText` をClaudeのコンテキストに読み込む
2. **▼内容** セクション以降を対象とする（▼概要は不要）
3. IT・AI・Claude関連トピックの**見出し一覧**と各トピックの**要点**をリストアップする

| 種類 | 扱い方 |
|---|---|
| トピック見出し | そのまま使用OK（固有名詞・ツール名は正確に） |
| 具体的な仕様・数値 | 正確に保持して言い換え |
| 学長の見解・推奨 | 「〜を推奨」「〜との見解」と明示して言い換え |
| 会員Q&A | 質問の趣旨と回答の要点のみ。個人名・固有情報は匿名化 |
| エピソード・事例 | 教訓や要点に圧縮。詳細描写は不要 |

---

### Step 5: IT・Claude・AI関連トピックを再構成する

**抽出対象キーワード（例）**
- Claude / Claude Code / Claude Cowork / Anthropic
- AI / ChatGPT / 生成AI / LLM / Gemini / Grok
- GitHub / プログラミング / コーディング / 自動化 / スクリプト
- Webデザイン / ホームページ / システム開発 / アプリ開発
- IT / テクノロジー / SaaS / DX / Cloudflare / セキュリティ

**出力フォーマット：**

```
【IT・AI関連トピック整理】○月○日 学長ライブより
━━━━━━━━━━━━━━━━━━

■ テーマ：（トピックの見出し）
━━━━━━━━━━━━━━━━━━
▶ 事実
（具体的な変更点・仕様・数値など。Claudeが言い換えて記述）

▶ 影響・意味
（自分の活動・学習にとって何を意味するか。Claudeが分析して記述）

▶ 補足（必要な場合のみ）
（背景情報・関連情報・注意点など）

…（該当トピック分くり返し）

━━━━━━━━━━━━━━━━━━
▼ Claudeのひとこと
（全体を通じて特に注目のポイントを1〜2文でコメント）

🔗 参照元: https://libecity.com/room_list?room_id=LIVE-Text
📅 取得日時: ○年○月○日
```

**注意：** 関連トピックが0件の場合は「今回のライブにはIT・AI関連のトピックはありませんでした」とチャットに表示して終了する。

---

### Step 6: Googleドライブにファイルとして保存する

1. ユーザーが設定したGoogle DriveフォルダIDを使って直接保存する

2. ファイル名:
   ```
   YYYY-MM-DD_学長ライブまとめ.md  ← 投稿タイトルから読み取ったライブ配信日
   ```
   **⚠️ 今日の日付ではなくライブ配信日を使う。**

3. **base64エンコード手順（Windows環境）：**
   ```bash
   node -e "
   const fs = require('fs');
   const content = \`（Step5の出力テキスト）\`;
   fs.writeFileSync(process.env.APPDATA + '/../Local/Temp/live_summary_b64.txt',
     Buffer.from(content, 'utf8').toString('base64'));
   console.log('done');
   "
   ```

4. 保存完了後、チャット画面に以下を表示する：
   ```
   ✅ ○月○日のまとめを保存しました
   抽出トピック数: ○件
   保存先: 学長ライブまとめ/YYYY-MM-DD_学長ライブまとめ.md
   ```

## エラー時の対応

| 状況 | 対応 |
|---|---|
| セッション切れでログイン画面にリダイレクト | `.env` から ID/PW を読み込んで再ログイン。成功後に `session.json` の `savedAt` を更新 |
| ログインフォームが見つからない | スナップショットを取得してフォーム構造を確認 |
| 「続きを読む」がクリックできない | `span.post_text_open` を `evaluate` で直接 `.click()` する |
| 【学長ライブ】投稿が見つからない | 投稿タイトル一覧をすべて取得し、「学長ライブ」を含む最新のものを選択する |
| Google Driveへの保存に失敗 | エラー内容をユーザーに伝え、整理済みの内容をチャット画面にそのまま表示する |

---

## 注意事項

- ログイン情報は `.env` で管理し、このスキルファイルには書かない
- `session.json` はGitに含めない（`.gitignore` で除外済み）
- 取得・保存するコンテンツは個人の学習目的に限定し、外部公開・転載はしない
- このスキルはClaude Code + Playwright MCP環境で動作する

# リベシティ 投稿フォーム自動入力スキル

## スキルの目的

`/libecity-check` で80点以上になった完成記事を、ノウハウ図書館の投稿フォームへ自動入力する。

**このスキルの責務：** フォームへの入力自動化のみ。タイトルの良し悪し・カテゴリ選定の深い分析は `/libecity`・`/libecity-check` が担う。

---

## 手順

### STEP 1 ｜ 引数を受け取る

`$ARGUMENTS` をファイル名として受け取る。

- **ファイル名が渡された場合**（例：`20260414_ClaudeCode体験_本文.md`）→ STEP 2へ
- **引数なしの場合** → 完成記事フォルダのファイル一覧を表示してユーザーに選択を促す

完成記事フォルダ：`~/Desktop/リベシティ/ノウハウ図書館/完成記事/`

フォルダが存在しない場合は自動的に作成する。

---

### STEP 2 ｜ 前提チェック

以下を確認し、問題があれば処理を止めて案内する：

| チェック内容 | 合格条件 | 不合格時の案内 |
|---|---|---|
| 完成記事ファイル | `_本文.md` ファイルが存在する | 「ファイルが見つかりません。パスを確認してください」 |
| チェックスコア | 同名 `_check.md` に「投稿OK」または「80点以上」の記載がある | 「`/libecity-check [ファイル名]` を先に実行してください（現在未実施 or スコア不足）」 |
| サムネイル画像 | 同名フォルダ内に `_thumbnail.png` が存在する | 「`/libecity-image [ファイル名]` でサムネイルを作成してください」 |

チェックファイルが存在しない場合はスコア不明として扱い、「チェックなしで投稿を続けますか？（リスクあり）」と確認する。

サムネイルが存在しない場合も同様に「サムネイルなしで続けますか？」と確認してから進む。

同名フォルダが存在しない場合は自動的に作成する。

---

### STEP 3 ｜ 投稿素材を1項目ずつ確認しながら生成する

記事ファイルを読み込み、①〜⑤を **1項目ずつ順番に提示・確認**する。
全項目の承認が得られたら STEP 4 へ進む。

修正を求められた場合は、その項目だけ再生成して再提示する。

---

#### ① タイトル最終確認・最適化

**現在のタイトルを表示したあと**、記事の性質（体験談型 or ハウツー型）を判断して、対応する型のブラッシュアップ案を2個提示する：

| 型 | 向いている記事 | 公式 | 実例 |
|---|---|---|---|
| **命令形型** | ハウツー・設定ガイド系 | 「これだけは〇〇しておいて！[ツール名/テーマ]ガイド」 | 「これだけは設定しておいて！Claude Code設定ガイド」 |
| **結果先行型** | 体験談・成長ストーリー系 | 「〇〇していたら、気づけば△△に。[属性]の軌跡」 | 「家族のために始めたYouTubeが、気づけば登録者17万人に」 |
| **対象読者型** | 初心者向け・比較・解説系 | 「[対象者]の味方、[ツール/手法]で[ベネフィット]する方法」 | 「初心者の味方、Claude Coworkで作業を任せる方法」 |

**共通ルール：**
- 文字数：35文字以内（サムネイルに収まる上限）
- 数字があれば必ず入れる（「17万人」「5分で」「3ステップ」）

---

#### ② カテゴリ選定（1つ選ぶ）

以下のフローで判定し、推薦カテゴリを1つ提示する：

```
副業・収益化・仕事効率化・AIで稼ぐ     → 稼ぐ力
節約・固定費・家計・ポイント活用       → 貯める力
投資・NISA・不動産・資産形成           → 増やす力
保険・リスク・セキュリティ・契約       → 守る力
体験・旅・消費・趣味・ライフスタイル   → 使う力
AI・IT・ツール・デジタルリテラシー     → ITリテラシー
コミュニティ・自己成長・マインド・その他 → その他
```

※AI活用・ClaudeCode・ツール解説・設定ガイド → **稼ぐ力 + ITリテラシー**（上位記事は2カテゴリ併用）

---

#### ③ タグ提案（3〜8個）

記事の内容から関連タグを提案する：

```
【確認済み：上位記事で実際に使われていたタグ】
AI系：    ClaudeCode / ClaudeCowork / Claude / AI活用 / 生成AI / AIエージェント / AIリテラシー
ツール系：設定 / 効率化
副業系：  副業 / YouTube / YouTuber / 動画編集 / ママ / 子育て

【推定タグ（未確認・要ユーザー確認）】
汎用系：  体験談 / やってみた / 初心者向け / 時短
お金系：  NISA / 節約 / 固定費 / フリーランス
```

タグ提示後に「実際のリベシティのタグ一覧と照らし合わせて調整してください」と一言添える。

---

#### ④ 要約文（150文字以内）

以下のテンプレートに記事の内容を当てはめて生成する：

```
[読者が共感できる悩み・状況を1文]。
[主人公（私）がやったこと・気づいたこと]で、[具体的な変化・結果]になりました。
[続きを読みたくなる一言 or 読者への問いかけ]
```

文字数をカウントして150文字以内に収まるか確認する。

---

#### ⑤ 本文の投稿用テキストを整形する

**変換ルール：**
| Markdown記法 | 処理 |
|---|---|
| `## 見出し` | そのまま |
| `### 見出し` | そのまま |
| `**太字**` | そのまま |
| `- 箇条書き` | そのまま |
| `\|表\|` | 箇条書きへの変換案を提示し承認を得る |
| `---` 区切り線 | 削除を提案する |
| コードブロック ` ``` ` | そのまま |

**変換が必要な箇所がある場合：** 変換箇所のみ抜粋して提示し、承認を得てから整形版を `_post_body.md` として保存する。

---

### STEP 4 ｜ 投稿素材をファイルに保存する

完成記事の同名フォルダ内に `_post_info.md` として保存する。

**保存先：**
`~/Desktop/リベシティ/ノウハウ図書館/完成記事/[ファイル名ベース]/[ファイル名ベース]_post_info.md`

**ファイル内容：**
```markdown
# 投稿情報シート

**記事ファイル：** [ファイル名]
**準備日：** YYYY-MM-DD

## ✅ タイトル（確定）
[確定タイトル]

## 📂 カテゴリ
[選定カテゴリ]

## 🏷️ タグ
[タグ1] [タグ2] [タグ3] ...

## 📝 要約（XXX文字）
[要約文]

## 🖼️ サムネイル
[ファイル名ベース]_thumbnail_vX.png

## 📄 本文（整形済み）
[ファイル名ベース]_post_body.md

## 🖼️ 挿絵配置（本文中の [↓ 画像をアップロード] 行に差し込む）
[挿絵がある場合のみ記載。なければ「なし」]

## 🔗 投稿先URL（編集中）
[編集URLを貼り付ける]

## 投稿ステータス
- [ ] 投稿完了
- [ ] 投稿後コメント確認（24時間以内）

```

### STEP 6 ｜ Playwright で投稿フォームを開く（任意）

「ブラウザで投稿フォームを開いて自動入力しますか？」とユーザーに確認する。

**Yes の場合：**

1. `mcp__playwright__browser_navigate` で投稿ページに遷移する
   - URL: `https://library.libecity.com/articles/new`

2. ログイン済みの場合 → 入力へ。ログイン画面が表示された場合 → 「ブラウザでログインしてから「続ける」と入力してください」と案内して待機

3. 以下の順で入力する：

   | 入力項目 | 操作 |
   |---|---|
   | タイトル | `browser_fill_form` でタイトル欄に入力 |
   | カテゴリ | `browser_select_option` または `browser_click` で選択 |
   | タグ | `browser_fill_form` でタグ欄に入力し Enter |
   | 要約 | `browser_fill_form` で要約欄に入力 |

4. 本文欄には自動入力しない。「`_post_body.md` の内容をコピーしてエディタに貼り付けてください」と案内して待機

5. 本文貼り付け完了後、下書き保存する

6. **投稿ボタンは押さない**。「プレビューで確認してからご自身で投稿ボタンを押してください」と案内して終了

---

### STEP 7 ｜ 優先順位まとめを更新する

`~/Desktop/リベシティ/ノウハウ図書館/記事案/00_優先順位まとめ.md`
を読み込み、該当記事の行に `📤投稿準備完了` を追記して上書き保存する。

---

---

## 注意事項

- **投稿ボタンは絶対に押さない**。最終確認はユーザーが行う
- STEP 3 は必ず1項目ずつ確認する。一括提示して「まとめて承認しますか？」はしない
- Playwrightでセッションがない場合はユーザーに手動ログインを依頼する（パスワードは絶対に入力しない）
- カテゴリは1つだけ選ぶ（複数選択不可）
- タグは8個を超えないようにする
